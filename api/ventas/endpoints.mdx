The Ventas API exposes five endpoints for managing sales records. All endpoints are served under the base path `/api/v1/ventas`. See the [overview page](/api/ventas/overview) for base URLs, the data model, and error handling.

---

## POST /api/v1/ventas

Creates a new sale record. Returns `201 Created` with the created object and a `Location` header pointing to the new resource.

### Body parameters

<ParamField body="direccionCompra" type="string" required>
  Purchase and delivery address. Cannot be blank.
</ParamField>

<ParamField body="valorCompra" type="int">
  Purchase value (amount).
</ParamField>

<ParamField body="fechaCompra" type="string" required>
  Purchase date in ISO-8601 format (`YYYY-MM-DD`). Cannot be null.
</ParamField>

<ParamField body="despachoGenerado" type="boolean" required default="false">
  Whether a dispatch record has been generated. Defaults to `false`.
</ParamField>

### Example

<CodeGroup>

```bash curl
curl --request POST \
  --url http://localhost:3001/api/v1/ventas \
  --header 'Content-Type: application/json' \
  --data '{
    "direccionCompra": "Av. Providencia 1234, Santiago",
    "valorCompra": 45000,
    "fechaCompra": "2024-03-15",
    "despachoGenerado": false
  }'
```

</CodeGroup>

**Request body**

```json
{
  "direccionCompra": "Av. Providencia 1234, Santiago",
  "valorCompra": 45000,
  "fechaCompra": "2024-03-15",
  "despachoGenerado": false
}
```

**Response body — 201 Created**

```json
{
  "idVenta": 1,
  "direccionCompra": "Av. Providencia 1234, Santiago",
  "valorCompra": 45000,
  "fechaCompra": "2024-03-15",
  "despachoGenerado": false
}
```

| Status | Description |
|--------|-------------|
| `201 Created` | Sale created successfully. `Location` header contains the URI of the new resource. |
| `400 Bad Request` | Validation failed. Required fields are missing or blank. |

## GET /api/v1/ventas

Returns a list of all sale records in the system.

### Example

<CodeGroup>

```bash curl
curl --request GET \
  --url http://localhost:3001/api/v1/ventas
```

</CodeGroup>

**Response body — 200 OK**

```json
[
  {
    "idVenta": 1,
    "direccionCompra": "Av. Providencia 1234, Santiago",
    "valorCompra": 45000,
    "fechaCompra": "2024-03-15",
    "despachoGenerado": false
  },
  {
    "idVenta": 2,
    "direccionCompra": "Calle Falsa 456, Valparaíso",
    "valorCompra": 28000,
    "fechaCompra": "2024-03-16",
    "despachoGenerado": true
  }
]
```

| Status | Description |
|--------|-------------|
| `200 OK` | Returns an array of all sales. Returns an empty array when no records exist. |

## GET /api/v1/ventas/{idVenta}

Returns the details of a single sale by its ID.

### Path parameters

<ParamField path="idVenta" type="Long" required>
  The unique identifier of the sale to retrieve.
</ParamField>

### Example

<CodeGroup>

```bash curl
curl --request GET \
  --url http://localhost:3001/api/v1/ventas/1
```

</CodeGroup>

**Response body — 200 OK**

```json
{
  "idVenta": 1,
  "direccionCompra": "Av. Providencia 1234, Santiago",
  "valorCompra": 45000,
  "fechaCompra": "2024-03-15",
  "despachoGenerado": false
}
```

| Status | Description |
|--------|-------------|
| `200 OK` | Sale found and returned. |
| `404 Not Found` | No sale exists with the given `idVenta`. |

## PUT /api/v1/ventas/{idVenta}

Updates an existing sale with new field values. All body fields are replaced with the provided values.

### Path parameters

<ParamField path="idVenta" type="Long" required>
  The unique identifier of the sale to update.
</ParamField>

### Body parameters

<ParamField body="direccionCompra" type="string" required>
  Updated purchase and delivery address. Cannot be blank.
</ParamField>

<ParamField body="valorCompra" type="int">
  Updated purchase value.
</ParamField>

<ParamField body="fechaCompra" type="string" required>
  Updated purchase date in ISO-8601 format (`YYYY-MM-DD`).
</ParamField>

<ParamField body="despachoGenerado" type="boolean" required>
  Updated dispatch status.
</ParamField>

### Example

<CodeGroup>

```bash curl
curl --request PUT \
  --url http://localhost:3001/api/v1/ventas/1 \
  --header 'Content-Type: application/json' \
  --data '{
    "direccionCompra": "Av. Providencia 1234, Santiago",
    "valorCompra": 50000,
    "fechaCompra": "2024-03-15",
    "despachoGenerado": true
  }'
```

</CodeGroup>

**Request body**

```json
{
  "direccionCompra": "Av. Providencia 1234, Santiago",
  "valorCompra": 50000,
  "fechaCompra": "2024-03-15",
  "despachoGenerado": true
}
```

**Response body — 200 OK**

```json
{
  "idVenta": 1,
  "direccionCompra": "Av. Providencia 1234, Santiago",
  "valorCompra": 50000,
  "fechaCompra": "2024-03-15",
  "despachoGenerado": true
}
```

| Status | Description |
|--------|-------------|
| `200 OK` | Sale updated successfully. Returns the updated object. |
| `400 Bad Request` | Validation failed on one or more fields. |
| `404 Not Found` | No sale exists with the given `idVenta`. |

---

## DELETE /api/v1/ventas/{idVenta}

Permanently deletes a sale record. Returns no body on success.

### Path parameters

<ParamField path="idVenta" type="Long" required>
  The unique identifier of the sale to delete.
</ParamField>

### Example

<CodeGroup>

```bash curl
curl --request DELETE \
  --url http://localhost:3001/api/v1/ventas/1
```

</CodeGroup>

| Status | Description |
|--------|-------------|
| `204 No Content` | Sale deleted successfully. No response body. |
| `404 Not Found` | No sale exists with the given `idVenta`. |
